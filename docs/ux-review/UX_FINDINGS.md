# HealthyFlow UX findings

Evidence was collected from the running application on 18 July 2026, the Lina demo workspace, DOM/accessibility snapshots, keyboard checks, measured control bounds, source inspection, and the existing test suite. Screenshot paths are relative to this directory. “Relevant files” identifies the implementation surface; no production UI files were changed.

| ID | Screen | Category | Finding | Evidence | Severity | Recommendation | Benefit | Effort | Relevant files |

| HF-001 | `/calories`, `/achievements` | Product IA / routing | Settings-gated routes cannot be reliably opened by URL or refresh. | On a cold Playwright navigation, both routes redirected to Today even though `GET /api/settings` returned both flags as `true`. `App` evaluates `settings?.flag` before `useSettings` finishes, while Workouts uses a different `?? true` fallback. | High | Add an explicit settings-loading gate. Preserve the requested location, then render the enabled route or a clear disabled-module state after settings resolve. | Bookmarks, refreshes, notifications, and future module deep links become dependable. | Small | `src/App.tsx`, `src/hooks/useSettings.ts` |
| HF-002 | `/add` | Product IA / modules | Add exposes Calories and Achievements even when the corresponding module is disabled. | With Calorie Intake turned off in the running demo, the Calories tab remained visible in [the disabled-module capture](screenshots/add-desktop-1440x900-calorie-module-disabled.png). A successful calorie add navigates to a route that then redirects away. | High | Build domain tabs from resolved settings, or remove the gates consistently if the approved product packaging makes health tracking core. | HF-003 | `/week` | Accessibility / design system | White theme makes core Week content unreadable. | Live White-theme capture shows white task titles and summary text disappearing against pale surfaces, while the shell changes correctly: [Week White theme](screenshots/week-desktop-1440x900-white-theme.png). | HF-007 | Today task rows | Accessibility / touch | The primary mobile Habit check-in control is only 16×16 px. | Playwright measured “Check in Drink water before coffee” at 16×16 px at 390×844, while the adjacent action menu was 44×44. | HF-009 | Mobile global navigation | Accessibility / navigation | The mobile drawer is visually modal but is not exposed or managed as a dialog. | The expanded state has a generic container, unnamed open and close buttons, no `aria-modal`, no focus trap, and the underlying main remains in the accessibility tree: [expanded drawer](screenshots/navigation-mobile-390x844-expanded.png). | High | Name both menu controls, use a labelled modal-navigation dialog or an equivalent inert drawer pattern, trap focus, close on Escape, and restore focus to the opener. | Gives mobile keyboard and assistive-technology users predictable navigation. | Medium | `src/components/Layout.tsx` |
| HF-010 | `/settings` | Accessibility / forms | Settings toggles have no accessible name, switch role, or checked state. | Accessibility snapshots list the feature and notification toggle buttons as unnamed buttons adjacent to headings. | HF-011 | `/settings` | Trust / interaction | Export Data and Delete Account are rendered as active destructive or privacy controls but have no behavior. | Both are buttons without `onClick` handlers in source and appeared actionable in [Settings](screenshots/settings-desktop-1440x900-default.png). | High | Disable them with an honest “Not available yet” explanation or implement the complete confirmed workflow. | HF-012 | `/settings` | Information architecture | Everyday preferences, AI billing, planning rhythm, modules, developer token scopes, and destructive account controls share one long equal-weight column. | Desktop capture is several viewport heights and mobile is an extreme stack: [desktop Settings](screenshots/settings-desktop-1440x900-default.png), [mobile Settings](screenshots/settings-mobile-390x844-default.png). | High | Add settings categories with local navigation: Account & Billing, Planning, Notifications, Health tools, Appearance, Connections/Advanced, and Data & Privacy. | HF-013 | `/calories`, `/workouts` | Interaction / date navigation | Health logs rely on a raw date input without Previous, Next, or Today actions. | Both live screens show only a date field.
