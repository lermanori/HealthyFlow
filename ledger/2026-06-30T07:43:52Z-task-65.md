# Task #65: Track and store calorie intake item usage for most-used and recent sorting

Added a `calorie_item` table with normalized-name indexing to track reusable calorie items independently of individual log entries. Every calorie entry now upserts its corresponding item and increments `usage_count` while updating `last_used_at`. Implemented service functions to query items sorted by usage (most-used) and by timestamp (recent), with unit tests verifying correct tracking and sort order. Logging snapshots nutrition values per entry without mutating canonical item defaults.
